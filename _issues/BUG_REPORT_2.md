# BUG REPORT: mutmut-win 1.0.7 — Trampoline KeyError during Clean Test Suite

**Reporter:** Claude Code Agent (pgm1980/nextgen-cot-mcp-server)
**Date:** 2026-04-10
**Version:** mutmut-win 1.0.7 (`5e33cdf1c83c915c4e0f4dbf0cb28a3de1f68940`)
**Severity:** Blocker (prevents all mutation testing)
**Environment:** Windows 11 Pro, Python 3.14.3, uv, hatchling build-backend

---

## Summary

The injected `_mutmut_trampoline()` function reads `os.environ['MUTANT_UNDER_TEST']` with a hard `__getitem__` lookup (no default/fallback). During the **clean test suite** phase, this environment variable is **not set**, causing every test that calls a mutated function to crash with `KeyError: 'MUTANT_UNDER_TEST'`.

The clean test suite runs **before** any mutation — it is supposed to verify that all tests pass on **unmodified** code. However, the source code in `mutants/src/` is already rewritten with trampoline instrumentation at this point, so the trampoline code executes and immediately fails.

---

## Steps to Reproduce

```bash
# 1. Any Python project with passing tests
uv run pytest  # all green

# 2. Run mutation testing
uv run mutmut-win run --paths-to-mutate src/nextgen_cot_mcp_server/infrastructure/identifiers.py

# 3. Output:
#   Running clean test suite...
#   Error: Clean test run failed with exit code 3. Fix tests before mutating.
```

---

## Root Cause

File: `mutants/src/nextgen_cot_mcp_server/infrastructure/identifiers.py` (auto-generated by mutmut-win)

```python
# Line 43-46 — injected by mutmut-win into EVERY mutated source file
def _mutmut_trampoline(orig, mutants, call_args, call_kwargs, self_arg=None):
    """Forward call to original or mutated function, depending on the environment"""
    import os
    mutant_under_test = os.environ['MUTANT_UNDER_TEST']  # <-- CRASH HERE
    if mutant_under_test == 'fail':
        from mutmut_win.__main__ import MutmutProgrammaticFailException
        raise MutmutProgrammaticFailException('Failed programmatically')
    elif mutant_under_test == 'stats':
        ...
```

The original function `uuid7()` is replaced with a trampoline wrapper:

```python
# Line 69-72 — original function renamed, trampoline installed
def uuid7() -> UUID:
    args = []
    kwargs = {}
    return _mutmut_trampoline(x_uuid7__mutmut_orig, x_uuid7__mutmut_mutants, args, kwargs, None)
```

During the clean test suite, `MUTANT_UNDER_TEST` is not in `os.environ`, so `os.environ['MUTANT_UNDER_TEST']` raises `KeyError`.

## Expected Behavior

The trampoline should fall through to the original function when `MUTANT_UNDER_TEST` is not set:

```python
def _mutmut_trampoline(orig, mutants, call_args, call_kwargs, self_arg=None):
    import os
    mutant_under_test = os.environ.get('MUTANT_UNDER_TEST')  # <-- .get() with None default
    if mutant_under_test is None:
        # Clean run — call original function directly
        result = orig(*call_args, **call_kwargs)
        return result
    if mutant_under_test == 'fail':
        ...
```

This would allow the clean test suite to pass (tests call the original functions through the trampoline), and only activate mutation dispatch when the environment variable is explicitly set.

---

## Workaround

None found. The trampoline code is injected into every mutated source file by mutmut-win's code rewriter. The user cannot modify the injected code without patching mutmut-win itself.

---

## Impact

- **All mutation testing is blocked** for any project with non-trivial functions
- Files with only constants (e.g., `sentinels.py`) work because no trampoline is injected (no function bodies to mutate)
- The `--dry-run` flag works correctly (4110 mutants counted) because it does not execute the mutated code
